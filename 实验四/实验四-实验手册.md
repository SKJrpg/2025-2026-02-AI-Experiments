## 1. 实验说明

## 1.1. 实验目标

在本地计算机上完整安装并配置 Ollama 大语言模型运行环境（安装包从 https://ollama.com/download 获取）

安装 OpenCode（安装方式见 https://opencode.ai），并将其连接到本地 Ollama 模型，形成可在终端中使用的 AI编程助手

阅读本地 OpenCode 中文说明文档，理解 OpenCode 的安装方式、Agent 模式和基础使用方法

使用 OpenCode 辅助撰写本次实验报告，理解本地 AI 编程助手的工作方式及其与传统编程工具的区别

## 1.2. 实验内容

1. 安装 Ollama，并确认本地服务能够正常运行

2. 拉取一个适合本机配置的本地模型，并完成命令行对话测试

3. 安装 OpenCode，阅读本实验目录下的 README.zh.md 中文说明文档

4. 使用 OpenCode 连接 Ollama，在实验目录中辅助生成实验报告内容

5. 在报告中说明 Ollama 与 OpenCode 的安装过程、OpenCode 的使用方法，以及使用 AI 辅助工具撰写报告的过程与体会

## 1.3. 实验要求

1. 必须完整记录 Ollama 的安装、模型下载、服务检查和接口测试过程

2. 必须完整记录 OpenCode 的安装、启动、模型选择和基础使用过程

3. 必须阅读本实验提供的 OpenCode 中文说明文档： README.zh.md

## 关于本地模型速度较慢的情况：

由于本地大模型的运行速度取决于电脑硬件配置，部分同学可能会遇到模型响应很慢的情况。如果本地模型速度不足以完成报告撰写，允许使用以下替代方案（但 Ollama 和 OpenCode 的安装、配置、测试过程仍必须完成）：

方案一：在 OpenCode 中连接云端模型（如 OpenCode Zen），继续使用 OpenCode 界面完成报告

方案二：使用豆包、ChatGPT、文心一言、Kimi 等 Web 端 AI 工具辅助撰写报告

采用替代方案的同学，须在实验报告的"遇到的问题与解决方法"中说明：选择了哪种方案、为什么选择（如本地模型响应速度过慢）、实际使用了哪些工具。

## 2.2. Ollama 安装与检查

Ollama 是一个可以在本地运行大语言模型的工具。安装完成后，它会在本机提供一个模型服务，默认地址为http://localhost:11434 。

[→] Ollama 安装包下载：https://ollama.com/download — 请根据你的操作系统选择对应的版本。更多模型和使用说明可访问 https://ollama.com。

## 2.2.1. Windows 安装

1. 打开 Ollama 官方下载页面：https://ollama.com/download

2. 下载 Windows 安装程序 OllamaSetup.exe

3. 双击安装程序并按提示完成安装

4. 安装完成后，打开 PowerShell 或 Windows Terminal，检查版本：

ollama -v 

## 2.2.2. macOS 安装

1. 打开 Ollama 官方下载页面：https://ollama.com/download

2. 下载 macOS 版本安装包

3. 将 Ollama 拖入 Applications 后启动

4. 打开终端检查版本：

ollama 

## 2.2.3. Linux 安装

Linux 可以使用官方安装脚本：

```shell
curl -fsSL https://ollama.com/install.sh | sh 
```

安装完成后检查版本：

```txt
ollama -v 
```

如果学校网络环境无法直接访问安装脚本，可以先在浏览器中打开 https://ollama.com/download 下载对应系统的安装包。

## 2.2.4. 检查 Ollama 服务

安装完成后，检查 Ollama 是否正常运行：

```txt
ollama list 
```

也可以在浏览器中访问：

```txt
http://localhost:11434 
```

如果服务正常，页面通常会显示：

```txt
Ollama is running 
```

也可以使用接口检查：

```batch
curl http://localhost:11434/api/tags 
```

## 2.5. 使用 Ollama 自动配置并启动 OpenCode

Ollama 提供了一条命令，可以自动配置并启动 OpenCode：

```txt
ollama launch opencode 
```

这条命令会做两件事：

1. 自动生成 OpenCode 连接 Ollama 所需的配置文件

2. 启动 OpenCode 并连接到本地的 Ollama 模型

启动后，OpenCode 会进入终端交互界面。使用 Tab 键切换 Agent，使用 /models 命令查看和选择可用的本地模型。

可以尝试输入：

你好，请介绍一下你能做什么。

如果 OpenCode 能够正常回应，说明本地模型已经与 OpenCode 联通。

## 2.6. 手动配置 OpenCode 连接 Ollama（备选方案）

如果 ollama launch opencode 无法正常工作，可以手动编写配置文件。手动配置的本质就是自己写一份"告诉OpenCode 去哪里找 Ollama、用什么模型"的文件。

## 2.6.1. 找到或创建配置文件

OpenCode 的全局配置文件位置：

```txt
# Windows
%USERPROFILE%\.config\opencode\opencode.json
# macOS / Linux
~/.config/opencode/opencode.json 
```

如果文件或目录不存在，请手动创建。可以先创建目录：

```powershell
# macOS / Linux
mkdir -p ~/.config/opencode

# Windows (PowerShell)
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.config\opencode" 
```

## 2.6.2. 编写配置文件

假设你已经通过 ollama pull qwen2.5:3b 下载了模型，配置文件内容如下：

```json
{
    "$schema": "https://opencode.ai/config.json",
    "provider": {
    "ollama": {
    "npm": "@ai-sdk/openai-compatible",
    "name": "Ollama (local)",
    "options": {
    "baseURL": "http://localhost:11434/v1"
    },
    "models": {
    "qwen2.5:3b": {
    "name": "Qwen2.5 3B"
    }
    }
    }
    }
} 
```


逐字段说明：


<table><tr><td>字段</td><td>含义</td></tr><tr><td>&quot;ollama&quot;</td><td>提供商的标识名,可以自定义</td></tr><tr><td>npm&quot;: &quot;@ai-sdk/openai-compatible&quot;</td><td>使用 OpenAI 兼容协议连接(Ollama 支持此协议)</td></tr><tr><td>&quot;baseURL&quot;: &quot;http://localhost:11434/v1&quot;</td><td>Ollama 服务的接口地址,注意末尾必须有 /v1</td></tr><tr><td>&quot;qwen2.5:3b&quot;</td><td>模型名称,必须与 ollama list 显示的名称完全一致</td></tr><tr><td>&quot;name&quot;: &quot;Qwen2.5 3B&quot;</td><td>在 OpenCode 界面中显示的名称,可以自定义</td></tr></table>

⚠ 常见错误： models 中的模型名称必须和 ollama list 看到的完全一致。如果你下载的是 qwen2.5:7b ，配置里也要写 qwen2.5:7b ，不能写 qwen2.5:3b 。

## 2.6.3. 启动并选择模型

配置完成后，在工作目录中启动 OpenCode：

opencode

如果 OpenCode 中看不到 Ollama 模型，先用 ollama list 确认模型名称，再检查 OpenCode 配置中的 models字段

如果本地模型回答较慢，属于正常现象，速度取决于电脑 CPU、GPU、内存和模型大小

如果本地模型速度确实太慢（例如等待超过 2-3 分钟才回复），可以改用 OpenCode 连接云端模型，或直接使用豆包、ChatGPT、文心一言、Kimi 等 Web 端 AI 工具辅助撰写报告。详见 1.3 节的说明
