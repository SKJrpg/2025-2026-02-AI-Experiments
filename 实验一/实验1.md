### 2 & 3. 统计图表生成代码（折线图与小提琴图）

您可以使用以下 Python 代码读取这三个 CSV 文件，并一键生成包含“**温度随时间变化折线图**”和“**温度分布小提琴图**”的对比图：

```python name=weather_analysis.py
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

# 1. 读取数据
df_bj = pd.read_csv('Beijing_filled.csv')
df_nj = pd.read_csv('Nanjing_filled.csv')
df_gz = pd.read_csv('Guangzhou_filled.csv')

# 提取时间格式并处理（将字符串转为datetime格式以便绘图）
df_bj['Time'] = pd.to_datetime(df_bj['Time'].astype(str), format='%Y%m%d%H')
df_nj['Time'] = pd.to_datetime(df_nj['Time'].astype(str), format='%Y%m%d%H')
df_gz['Time'] = pd.to_datetime(df_gz['Time'].astype(str), format='%Y%m%d%H')

# 增加城市标签并合并数据
df_bj['City'] = 'Beijing'
df_nj['City'] = 'Nanjing'
df_gz['City'] = 'Guangzhou'
df_all = pd.concat([df_bj, df_nj, df_gz])

# 设置中文字体（根据您的系统可能需要微调字体名称）
plt.rcParams['font.sans-serif'] = ['SimHei']  # Windows用黑体
plt.rcParams['axes.unicode_minus'] = False 

# 创建画布 (1行2列)
fig, axes = plt.subplots(2, 1, figsize=(14, 12))

# --- 图1: 温度随时间变化折线图 ---
sns.lineplot(data=df_all, x='Time', y='TMP', hue='City', 
             palette={'Beijing':'#1f77b4', 'Nanjing':'#ff7f0e', 'Guangzhou':'#2ca02c'}, 
             ax=axes[0])
axes[0].set_title('北京、南京、广州一周逐小时气温变化曲线', fontsize=16)
axes[0].set_xlabel('日期与时间', fontsize=12)
axes[0].set_ylabel('温度 (℃)', fontsize=12)
axes[0].grid(True, linestyle='--', alpha=0.6)

# --- 图2: 温度分布小提琴图 ---
sns.violinplot(data=df_all, x='City', y='TMP', 
               palette={'Beijing':'#1f77b4', 'Nanjing':'#ff7f0e', 'Guangzhou':'#2ca02c'}, 
               inner='quartile', ax=axes[1])
axes[1].set_title('北京、南京、广州气温分布小提琴图', fontsize=16)
axes[1].set_xlabel('城市', fontsize=12)
axes[1].set_ylabel('温度 (℃)', fontsize=12)

plt.tight_layout()
plt.show()
```

Q1:大模型可能依据什么原则判断温度数据为异常值？（如气温的合理范围、数据的
突变程度等）



大模型（以及数据科学家在处理此类数据时）判断温度数据为异常值，通常会依据以下几个核心原则和逻辑：

**1. 物理常识与业务规则（合理阈值范围）**
这是最直观的判断标准。模型会结合地理位置、季节和气候特征来判定：

*   **绝对物理极限：** 地球自然环境下的气温通常在 -90℃ 到 60℃ 之间。数据中出现的 `100.0℃`（水沸腾的温度）或 `-100.0℃` 显然违背了地球的常规自然规律。
*   **特定时空背景：** 结合本题背景“北京的12月”，北京初冬的气温通常在 -15℃ 到 15℃ 之间。如果数据在这个时期出现了 30℃ 或者 -40℃，即便在地球上可能存在，但在此时此地属于异常。

**2. 时间序列的连续性与突变程度（变化速率）**
气温变化具有物理上的热惯性，是一个渐变的过程，短时间内的剧烈波动通常是不合理的：

*   **单小时突变限制：** 正常情况下，自然环境气温在一小时内的升降幅度很少超过 5℃-10℃。如果上一小时是 -3℃，下一小时突然变成 100℃，随后又变回 -4℃，这种极其陡峭的“毛刺（Spike）”违背了热力学规律，极大可能是传感器故障或数据传输错误。

**3. 常见的传感器错误代码特征（特定数值）**
在气象和物联网设备采集中，当传感器损坏、断电或超出量程时，系统往往会自动填充一些特定的默认“错误码”：

*   常见错误码如：`9999`、`-99.9`、`100`、`-100` 等。模型识别到这些非常规的整数或极端阈值时，会优先怀疑其为系统默认生成的缺失/异常标识，而非真实观测值。

**4. 统计学分布特征**
模型如果进行定量分析，还会隐含地使用统计学方法：

*   **3σ 原则（标准差）：** 如果某条数据偏离了整体数据平均值的 3 个标准差以上，就属于极小概率事件，大概率为异常。
*   **箱线图原理（IQR）：** 判断数据是否超出了上下四分位数的合理范围。

综上所述，在您提供的北京12月温度数据中，大模型主要是结合了**“气候常识（北京冬天不可能有100度）”**、**“连续性断层（一小时内温度不可能突变100多度）”**以及**“典型的传感器错误特征（恰好是100和-100这样的极端整数）”**这三个原则，准确识别出了异常值。



Q2:针对逐小时气温数据，哪种缺失值填充方式更合理？大模型的填充结果是否符合
该原则？



针对逐小时气温数据，最合理的缺失值填充方式是基于时间序列连续性的**插值法（尤其是线性插值或样条插值）**。大模型（在此次任务中）的填充结果**完全符合这一原则**。

以下是具体的原理解释及结果验证：

### 1. 为什么“线性插值”针对逐小时气温最合理？

* **符合热力学物理规律（连续性与渐变性）：** 气温是一个受大气热容影响的连续物理量。在短短几个小时内，如果没有极端天气突变，温度的升降一定是一个平滑的渐变过程。它不会像股票价格那样发生瞬时的“断崖式跳空”。
* **规避了常规统计填充的致命缺陷：**
  * **均值/中位数填充（不合理）：** 如果缺失的是凌晨的低温数据，用全天的平均温度去填，会导致凌晨温度突发性偏高，完全破坏了气温的日变化规律。
  * **向前/向后填充法（不合理）：** 会让数据呈现“阶梯状”的死水状态，不符合温度时刻在动态变化的自然现实。
* **线性插值的优势：** 它在缺失值前后的两个已知时间点之间拉一条直线，按时间比例均匀分配温度变化。对于1~3小时的短时间缺失，这种方法极其逼近真实的自然温度变化曲线。

### 2. 大模型的填充结果是否符合该原则？

**完全符合。** 在此前的操作中，大模型严格采用了**线性插值法**，并在首尾两端采用了最近邻填充。我们可以通过此前数据中的具体案例来验证其合理性：

**数据验证案例（以北京 12月2日 数据为例）：**
* `2019120212` (中午12点) 真实温度：**4.0℃**
* `2019120213` (下午13点) 原始缺失
* `2019120214` (下午14点) 原始缺失
* `2019120215` (下午15点) 真实温度：**3.0℃**

在这长达3个小时的跨度中，温度从 4.0℃ 降到了 3.0℃。大模型计算出的填充结果为：
* `2019120213` 填充为：**3.7℃**
* `2019120214` 填充为：**3.3℃**

**符合性评价：**
这个填充结果极其完美地还原了真实的物理场景——午后气温开始缓慢、均匀地下滑（每小时下降0.33度左右）。如果在折线图上观察，12点到15点之间依然是一条平滑的曲线，没有任何突兀的“毛刺”或“断层”。

**总结：** 对于逐小时气象数据，大模型摒弃了粗暴的整体均值填充，选择了尊重时间维度特征的线性插值法，这在数据科学和气象学处理规范中是**标准且最合理**的做法。
